# LCK Archive Prototype

이 저장소는 LCK 아카이브 웹사이트의 초기 프로토타입입니다. 주간 경기 일정과 팀 로스터를 중심으로, 추후 통계/연혁 정보를 확장하기 위한 기본 레이아웃을 제공합니다.

## 새 맥북에서 개발 환경 준비하기
맥OS에 기본으로 설치되어 있지 않은 도구를 먼저 준비해야 합니다. 아래 순서를 그대로 따라 하면 됩니다.

### 1. 터미널 열기
1. `Command + Space` 를 눌러 Spotlight를 엽니다.
2. "Terminal"을 검색한 뒤 엔터를 눌러 실행합니다.

### 2. Homebrew 설치 (패키지 관리자)
1. 아래 명령을 **그대로 복사**해 터미널에 붙여넣고 엔터를 누릅니다.
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
2. 처음 실행 시 "Command Line Tools" 설치 안내가 뜨면 **Install(설치)** 를 눌러 완료될 때까지 기다립니다. (10분 이상 걸릴 수 있습니다.)
3. "Password:" 가 보이면 **화면에 글자가 보이지 않아도** 맥 로그인 비밀번호를 입력하고 엔터를 누르세요.
4. 마지막에 `Press RETURN/ENTER to continue` 메시지가 나오면 엔터를 눌러 마무리합니다.
5. 설치가 끝난 뒤 아래 명령으로 정상 설치 여부를 확인합니다.
   ```bash
   brew --version
   ```
   버전 번호가 출력되면 성공입니다.

### 3. Git & Node.js 설치
```bash
brew install git node
```
- **Git**: 버전 관리를 위한 도구입니다.
- **Node.js**: 로컬 서버 실행, 향후 빌드 자동화를 위해 필요합니다.

### 4. 저장소 가져오기
GitHub 계정을 사용해 이 저장소를 Fork(복제)했다면, 아래처럼 본인 계정으로 클론합니다. 아직 Fork하지 않았다면 GitHub에서 "Fork" 버튼을 눌러 개인 저장소를 만든 뒤 진행하세요.
```bash
cd ~/Desktop
git clone https://github.com/<your-id>/LJE.git
cd LJE
```
> `<your-id>`를 본인 GitHub 아이디로 바꿔 적으세요.

### 5. 개발용 서버 실행
정적 파일만 있는 프로젝트지만, 다중 파일 구조를 확인하려면 간단한 서버를 띄우는 것이 편합니다.
```bash
npx serve .
```
명령을 실행하면 `http://localhost:3000`과 같은 주소가 나오며, 브라우저에서 접속하면 현재 작업 중인 화면을 볼 수 있습니다. 서버를 종료하려면 터미널에서 `Ctrl + C`를 누르세요.

### 6. 자주 묻는 질문 & 트러블슈팅
- **`chsh: no changes made` 라는 문구가 나와요.** → macOS Catalina 이상은 기본 쉘이 이미 `zsh`이므로 무시해도 됩니다.
- **명령 실행 후 아무 것도 안 되는 것 같아요.** → 설치 작업 중에는 추가 출력이 없을 수 있습니다. 설치가 끝날 때까지 기다렸다가 다음 안내가 나오면 따라 주세요.
- **비밀번호를 입력했는데 화면에 아무 것도 안 보여요.** → 터미널에서 비밀번호는 보이지 않는 것이 정상입니다. 입력 후 엔터를 누르면 됩니다.
- **설치 도중 창을 닫았어요.** → 다시 터미널을 열어 2번 단계부터 반복하면 됩니다.

## 프로젝트 구조
```
LJE/
├── assets/             # 이미지, 아이콘 등 정적 리소스 보관용 (현재 비어 있음)
├── index.html          # 메인 페이지
├── scripts/
│   └── main.js         # 내비게이션 하이라이트 등 인터랙션 스크립트
├── styles/
│   └── main.css        # 전체 스타일 정의
└── README.md           # 이 문서
```

## 앞으로 필요한 자료 요청
프로토타입을 실제 서비스로 확장하려면 아래 자료가 필요합니다. 가능한 범위에서 수집해 제공해 주세요.

1. **경기 일정 데이터**: 시즌·스플릿·주차·경기 시간·장소·승패 등 공식 소스를 기반으로 한 구조화된 정보(CSV 또는 Google Sheet 권장)
2. **팀 & 선수 정보**: 각 팀의 로스터, 코치진, 선수 프로필 이미지, 주요 기록(최근 시즌 기준)
3. **연혁 콘텐츠**: 명경기 영상 링크, 시즌별 하이라이트, 주요 이벤트 타임라인
4. **브랜딩 에셋**: 로고, 팀 컬러 코드, 폰트 사용 가이드 등 디자인 자산

자료를 정리해 전달해 주시면, 해당 데이터를 바탕으로 UI 컴포넌트와 API 연동 구조를 설계해 드리겠습니다.

## 배포 전 점검 체크리스트
- [ ] `index.html`에서 모든 링크와 이미지 경로가 올바른지 확인
- [ ] `styles/main.css`가 정상적으로 로드되는지 브라우저 개발자 도구(Network 탭)에서 확인
- [ ] 모바일 화면(DevTools Device Toolbar)에서도 레이아웃이 깨지지 않는지 검토
- [ ] 추후 자동 배포(Vercel, Netlify 등)를 사용할 경우, 루트 디렉터리(`index.html`)가 배포 대상인지 설정

필요한 추가 지원이 있다면 언제든 요청해 주세요.
